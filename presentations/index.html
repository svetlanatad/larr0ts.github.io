Online Classes Management System

Phase 1: ERD and Relational Schema

By Svetlana Tadevosyan
Date: 17 November, 2024

Project Overview

The Online Classes Management System is a database project to manage online courses, students, teachers, and enrollments.

Entities: - Students - Teachers - Classes - Enrollments

Focus: - Database design - MySQL operations - Scalability and efficiency

Relational Schema

The Relational Schema displays the relationships among: - Students, Teachers, Classes, and Enrollments - Students are related to Enrollments (1-to-many) - Teachers are related to Classes (1-to-many) - Classes are related to Enrollments (1-to-many)

Relational Schema

Students Table:

  • student_id (Primary Key)
  • name
  • email
  • status

Teachers Table:

  • teacher_id (Primary Key)
  • name
  • email

Classes Table:

  • class_id (Primary Key)
  • class_name
  • description
  • teacher_id (Foreign Key)

Enrollments Table:

  • enrollment_id (Primary Key)
  • student_id (Foreign Key)
  • class_id (Foreign Key)

SQL Schema (DDL)

-- Create Students Table
CREATE TABLE Students (
    student_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    status VARCHAR(20) NOT NULL
);

-- Create Teachers Table
CREATE TABLE Teachers (
    teacher_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL
);

-- Create Classes Table
CREATE TABLE Classes (
    class_id INT AUTO_INCREMENT PRIMARY KEY,
    class_name VARCHAR(100) NOT NULL,
    description TEXT,
    teacher_id INT,
    FOREIGN KEY (teacher_id) REFERENCES Teachers(teacher_id)
);

-- Create Enrollments Table
CREATE TABLE Enrollments (
    enrollment_id INT AUTO_INCREMENT PRIMARY KEY,
    student_id INT,
    class_id INT,
    FOREIGN KEY (student_id) REFERENCES Students(student_id),
    FOREIGN KEY (class_id) REFERENCES Classes(class_id)
);

Conclusion

The Online Classes Management System database design lays a solid foundation for managing students, teachers, classes, and enrollments within an educational platform. With an efficiently structured ERD and normalized relational schema, this design ensures data integrity, scalability, and ease of querying, which are essential for any modern online education system.
